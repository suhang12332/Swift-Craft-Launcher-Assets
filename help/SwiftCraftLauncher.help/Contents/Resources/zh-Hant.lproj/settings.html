啟動器設置

Swift Craft Launcher 提供了多種配置選項，讓您自定義使用體驗。

通用設置

語言選擇

選擇應用語言。更改後需要重啟應用才能生效。

支援的語言：阿拉伯語 (ar)、丹麥語 (da)、德語 (de)、英語 (en)、西班牙語 (es)、芬蘭語 (fi)、法語 (fr)、印地語 (hi)、義大利語 (it)、日語 (ja)、韓語 (ko)、挪威語 (nb)、荷蘭語 (nl)、波蘭語 (pl)、葡萄牙語 (pt)、俄語 (ru)、瑞典語 (sv)、泰語 (th)、土耳其語 (tr)、越南語 (vi)、簡體中文 (zh-Hans)、繁體中文 (zh-Hant)。

主題選擇

選擇主題模式：系統、淺色或深色。

介面佈局風格

設定啟動器介面佈局風格。可選項：classic 和 focused。

classic 適合較傳統的多區域佈局；focused 更強調內容聚焦與簡潔體驗。

啟動器工作目錄

設置啟動器工作目錄。啟動器管理的所有遊戲檔案都將放在此資料夾中。可以重置為預設值。

【重要】更改工作目錄後，原始路徑中的遊戲將不再出現在「遊戲列表」中。建議轉移或重新下載它們。最好在首次打開啟動器時更改工作目錄。

並發下載數量

設置並發下載數量，範圍：1-64。更高的並發性可以提高下載速度，但也會帶來更大的性能開銷。

GitHub 代理設置

配置 GitHub 代理設置，自動為 GitHub 相關的 URL 添加代理前綴，幫助解決某些地區訪問 GitHub 資源困難的問題。

啟用/停用開關：控制是否對 GitHub URL 應用代理。預設：啟用。

代理 URL：配置代理伺服器地址。預設：https://gh-proxy.com。代理 URL 必須是有效的 URL，協定為 http:// 或 https://。

自動應用：啟動器會自動對以下 GitHub 網域應用代理：

  • github.com - GitHub 倉庫頁面和發布版本

  • raw.githubusercontent.com - 原始檔案內容

排除網域：以下網域不應用代理：

  • api.github.com - GitHub API 端點（通常無需代理即可訪問）

使用場景：

  • GitHub API 調用（最新版本、貢獻者、更新檢查等）

  • 從 GitHub 倉庫下載檔案

  • 整合包依賴下載

  • Sparkle 更新服務下載

【註】啟用後，代理會自動應用，無需手動配置每個 URL。如果預設代理不可用，可以更換為其他 GitHub 代理服務，例如 https://ghproxy.com 或 https://mirror.ghproxy.com。

通用 Sheet 高度限制

設定啟動器中通用 Sheet 彈窗的最大高度限制，用於控制彈出面板在不同螢幕尺寸下的顯示高度。

如果您希望彈窗顯示更多內容，可以適度提高該值；如果希望介面更緊湊，則可以降低該值。

遊戲設置

預設API源

選擇預設的遊戲資源平台資料來源。可用選項：Modrinth 和 CurseForge。

顯示快照版

控制遊戲版本列表中是否包含快照版。

啟用後，建立遊戲或選擇版本時會顯示實驗性快照版本；關閉後，僅顯示一般正式版本。

AI崩潰分析

啟用/停用AI崩潰分析功能。此功能需要先成功配置AI設置。

預設記憶體分配

設置全域預設記憶體範圍（Xms-Xmx）。

同步語言到 options.txt

控制在建立新遊戲時，是否將啟動器目前語言自動同步到遊戲的 options.txt。

啟用後，新建立的遊戲實例會盡量預設使用與啟動器一致的語言設定。

遊戲資源資訊

顯示快取檔案數量和總大小（唯讀）。

清理快取

用於清理啟動器快取的遊戲資源檔案。

執行後可以釋放磁碟空間，但後續再次存取相關資源時，啟動器可能需要重新下載對應內容。

遊戲高級設置

遊戲高級設置為單個遊戲實例的獨立配置，只有在已選取某個遊戲後，對應標籤頁才會可用。

Java 路徑

自定義Java可執行檔案路徑，可重置為預設值。

以下是 MC 版本及其最低要求的 Java 版本：

  • 1.12 - Java 8

  • 1.17 - Java 16

  • 1.18 - Java 17

  • 1.20.5 - Java 21 (64-bit)

垃圾回收器

選擇垃圾回收器類型，根據Java版本自動過濾可用選項。

可用的垃圾回收器選項：

  • G1GC - Java 7+

  • ZGC - Java 11+

  • Shenandoah - Java 12+

  • Parallel - Java 1.0+

  • Serial - Java 1.0+

性能優化預設

選擇性能優化級別：

  • 停用 - 不啟用任何優化

  • 基礎 - 啟用基本優化

  • 平衡 - 啟用記憶體和執行緒優化（推薦）

  • 最大 - 啟用所有優化包括Aikar Flags（僅G1GC可用）

記憶體範圍

為目前遊戲單獨設定 Xms / Xmx 記憶體範圍（512 MB - 最大記憶體）。

此設定會覆蓋全域預設記憶體分配，僅對目前選取的遊戲生效。

自定義JVM參數

手動輸入JVM參數（與垃圾回收器和性能優化互斥）。

環境變數

設置遊戲運行時的環境變數。

AI設置

Swift Craft Launcher 整合了 AI 助手功能，支援多種 AI 提供商，可用於遊戲崩潰分析、問題解答等場景。

AI提供商選擇

選擇AI服務提供商。啟動器支援多個提供商，提供統一介面：

  • OpenAI - 預設 API URL：https://api.openai.com，API 路徑：/v1/chat/completions。支援自定義 API URL（相容 DeepSeek 等 OpenAI 格式服務）。使用 Bearer Token 認證。

  • Ollama - 本地部署選項。預設 API URL：http://localhost:11434，API 路徑：/api/chat。支援自定義 Base URL 和遠端 Ollama 伺服器。API Key 可選。

API金鑰

輸入所選提供商的 API 金鑰。輸入框支援顯示/隱藏切換（眼睛圖示）以確保安全。這是必填項，發送訊息前會進行驗證。

為提高安全性，API Key 不儲存在一般設定中，而是保存在系統 Keychain 中。

【重要】API 金鑰是 AI 功能的必需項。請確保妥善保管，不要與他人分享。

API URL（OpenAI 格式）

僅在選擇 OpenAI 格式提供商時顯示。允許自定義 API URL，以相容 DeepSeek 等 OpenAI 格式相容服務。如果留空，將使用提供商的預設 URL。

Ollama Base URL

僅在選擇 Ollama 時顯示。設置 Ollama 服務地址。預設：http://localhost:11434。如果配置了遠端 Ollama 伺服器，也支援使用。

模型名稱

輸入要使用的 AI 模型名稱。這是必填項，發送訊息前必須填寫。首尾空白字元會自動去除。

【重要】模型名稱必須與所選提供商提供的模型匹配。例如，OpenAI 使用 "gpt-4" 或 "gpt-3.5-turbo" 等模型，而 Ollama 使用您本地安裝的模型名稱。

AI頭像URL

設置 AI 頭像的 URL 地址。支援使用 MinecraftSkinUtils 元件進行即時預覽（預覽尺寸：42x42）。預設：https://mcskins.top/assets/snippets/download/skin.php?n=7050

AI功能

AI崩潰分析

啟用後，當檢測到遊戲崩潰時，啟動器會自動收集崩潰日誌並發送到 AI 視窗進行分析。

工作流程：

  1. 崩潰檢測：啟動器監控遊戲進程退出碼，並檢查最近 5 分鐘內生成的崩潰報告。

  2. 日誌收集：優先收集 crash-reports 資料夾中的所有檔案。如果未找到崩潰報告，則收集 logs/latest.log。

  3. AI分析：AI 聊天視窗自動打開，並附帶收集的日誌檔案，允許您與 AI 互動以獲取崩潰分析和解決方案建議。

【註】此功能需要先成功配置 AI 設置。此功能的開關位於遊戲設置部分。

AI聊天功能

AI 聊天功能支援：

  • 流式響應：即時流式輸出（SSE 格式），提供即時回饋

  • 訊息歷史：在整個會話過程中維護對話上下文

  • 檔案附件：支援發送文字檔案進行分析（≤100KB 的檔案會被讀取並發送，更大的檔案僅顯示檔案資訊）

  • 文字截斷：超過 5000 字元的内容會自動截斷

使用場景：

  • 遊戲崩潰分析（自動或手動）

  • 問題諮詢和故障排除

  • 日誌檔案和配置檔案分析

玩家設置

玩家設置用於管理登入方式、皮膚站與認證服務等與帳號相關的行為。

離線登入

控制是否啟用離線登入功能。

啟用後，可以在不使用正版帳號登入的情況下建立和使用離線玩家檔案。

預設皮膚站 / Yggdrasil 伺服器

設定預設使用的皮膚站或 Yggdrasil 認證伺服器位址。

適用於需要接入第三方認證系統的場景，便於統一管理玩家檔案的登入來源。

authlib-injector

玩家設置頁會顯示 authlib-injector 的下載與狀態資訊。

當使用 Yggdrasil 等第三方認證服務時，啟動器可藉由 authlib-injector 提供相容支援。
